Learn how to use PugTail from the command line

Complete guide to PugTail CLI options, configuration, and best practices

üì¶Installation

Install PugTail via npm or yarn

Using npm:

npm install pug-tail@alpha

Using yarn:

yarn add pug-tail@alpha

üöÄBasic Usage

Simple commands to get started

Compile a single file:

npx pug-tail src/index.pug -o dist/index.html

Compile a directory:

npx pug-tail src -o dist

Using a config file:

npx pug-tail -c pugtail.config.js

‚öôÔ∏èCLI Options

Main command-line options

-o, --out <dir>

Output directory or file path

-w, --watch

Watch mode - recompile on file changes

-c, --config <path>

Path to config file (pugtail.config.js)

--obj <path>

Data file (JSON/YAML) to inject into templates

--pretty

Pretty-print the output HTML

--basedir <path>

Base directory for absolute includes

-d, --debug

Enable debug output for troubleshooting

üìùConfig File

Using pugtail.config.js for project settings

Example pugtail.config.js:

export default {
  files: {
    input: 'src/**/*.pug',
    output: 'dist',
    root: 'src/pages'
  },
  basedir: '.',
  pretty: true,
  data: './data/global.json'
}

üìå Notes:

  • Config file paths are relative to the config file location
  • CLI options override config file settings
  • Supports both .js and .mjs extensions

üíæData Loading

Two ways to load JSON data into your templates

1. Global data via config:

// pugtail.config.js
export default {
  data: './data/global.json'
}

Available in all templates

2. Per-page data via $dataFiles:

//- pages/index.pug
- const $dataFiles = ['/data/page.json']

h1= pageTitle

Only available in entry files (not included components)

üí° Tips:

  • Combine both methods: global data for site-wide settings, $dataFiles for page-specific content
  • Absolute paths in $dataFiles (starting with /) are resolved from basedir
  • Relative paths are resolved from the entry file location

üëÅÔ∏èWatch Mode

Automatic recompilation during development

Enable watch mode:

npx pug-tail src -o dist -w

Watch with config:

npx pug-tail -c pugtail.config.js -w

Features:

  • Monitors file changes in real-time
  • Automatically recompiles on save
  • Debounced to prevent multiple rebuilds
  • Press Ctrl+C to stop watching

üí°Examples

Common use cases and patterns

Static site with global navigation:

  1. Create data/global.json with navigation data
  2. Set data: './data/global.json' in config
  3. Use navigation variable in Layout component

Blog with per-post metadata:

  1. Create data/posts/post-1.json for each post
  2. Use $dataFiles in each post template
  3. Access post-specific data like title, date, author

Multi-language site:

  1. Create data/en.json and data/ja.json
  2. Use $dataFiles to load language-specific content
  3. Generate separate HTML files for each language

‚ú®Tips & Tricks

Best practices for using PugTail

Organize components:

Keep components in a dedicated folder like components/ and use include to load them

Use glob patterns:

In config files, use patterns like **/*.pug to match all .pug files recursively

Separate data concerns:

Use global.json for site-wide data (navigation, footer), page-specific JSON for content

Debug with --debug:

When things don't work as expected, use the -d flag to see detailed processing information

Pretty print for development:

Use --pretty flag during development for readable HTML output
